# 10 — UI / Design System: "CRT Sommelier"

The entire app is a dark terminal: black, monospace, phosphor bloom, Blade Runner
CMYK accents, Vercel-grade restraint. This is a *committed* aesthetic — there is no
light mode. Tokens below are pinned; do not substitute a generic look.

## Design tokens (define as CSS variables in globals.css, use via Tailwind)

| Token | Value | Use |
|---|---|---|
| `--bg` | `#050505` | page background |
| `--bg-raised` | `#0A0A0B` | panels, cards |
| `--border` | `#1A1A1C` | 1px hairlines everywhere |
| `--text` | `#E8E8E3` | body (warm phosphor white) |
| `--text-dim` | `#8A8A85` | secondary text |
| `--cyan` | `#00E5FF` | primary accent: links, active states, the tutor's voice |
| `--magenta` | `#FF2E88` | destructive, "disliked", alerts |
| `--amber` | `#FFB000` | ratings, "mixed", highlights, wine-warm accents |
| `--green` | `#3FB950` | success, "liked" |

- Type: **monospace everywhere** — Geist Mono via `next/font` (fallback JetBrains Mono,
  ui-monospace). No serif, no sans anywhere.
- Corners: `0px` radius (2px max on inputs). Borders 1px `--border`; accent borders on
  focus/active.
- Spacing: generous. Vercel restraint — the glow is the decoration; layouts stay quiet.

## Signature effects (tasteful, performance-cheap)

- **Bloom**: accent-colored text/borders carry a soft same-color glow
  (`text-shadow: 0 0 8px` at ~35% alpha; one utility class per accent). Body text never
  glows.
- **Scanlines**: a fixed, full-viewport overlay — 2px repeating horizontal lines at
  ~3% white opacity, `pointer-events: none`. Subtle enough that screenshots read
  "expensive," not "gimmick."
- **Cursor**: the chat input renders a blinking block cursor `▮`; streaming assistant
  text ends with one while in flight.
- **Prompt markers**: user messages prefixed `>` (cyan), assistant messages prefixed
  `somm@cellar:~$` or a simpler `$` marker — pick one and keep it consistent.
- **ASCII structure**: section headers styled like terminal chrome (e.g.
  `── TASTING JOURNAL ──`), tables/cards may use box-drawing characters. Ratings render
  as block glyphs `▮▮▮▮▯`.
- All effects respect `prefers-reduced-motion` (no blink/animation) and none may sit
  above interactive elements.

## Accessibility floor

Body text contrast ≥ 4.5:1 against `--bg` (the tokens above satisfy this — keep it
true if tuning). Focus states visible (accent border + glow). Semantic HTML;
keyboard-navigable chat and forms.

## Profile identity (specs/11)

- Each household profile owns one accent color from the four (cyan, magenta, amber,
  green). The profile's name renders in its accent (with bloom) everywhere it appears:
  picker tiles, shell switcher, journal author badges, shared-tasting score rows,
  "for the table" cards.
- The profile picker (`/profiles`) is a full-screen terminal panel: `WHO'S TASTING?`
  header, one bordered tile per profile (name bloomed in its accent), `+ NEW TASTER`.

## App shell

- Left rail (or top bar on mobile): DASHBOARD / CHAT / JOURNAL / GRAPES / PROFILE +
  logout, styled as a terminal menu with the active item cyan-bloomed.
- The rail footer shows the **active profile's name in its accent color**; clicking it
  returns to the picker (specs/11).
- Route pages share the shell; `/`, `/login`, `/signup`, `/profiles` are minimal
  full-screen terminal panels (the landing page is a single styled panel with tagline +
  login/signup — no marketing site).

## Acceptance criteria

- **AC-UI-1**: globals define the token variables above and the app renders on `--bg`
  black with monospace type throughout (e2e asserts computed body background + font
  family).
- **AC-UI-2**: The scanline overlay exists, is non-interactive (`pointer-events:
  none`), and animations honor `prefers-reduced-motion` (e2e / component test).
- **AC-UI-3**: Verdict badges use the pinned semantic colors (liked=green,
  mixed=amber, disliked=magenta) (component/e2e).
- **AC-UI-4**: Nav shell reaches all five sections and marks the active route (e2e).
- **AC-UI-5**: The shell shows the active profile's name styled with its accent color,
  and two profiles in a household render with distinct accents (e2e).
